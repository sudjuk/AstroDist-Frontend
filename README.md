# Лабораторная 6

## Цель работы

Внедрить адаптивность, PWA-функциональность и развернуть приложение.

## Порядок показа

1. Открыть GitHub Pages с mock на телефоне и сохранить PWA.
2. Запустить PWA, применить фильтрацию услуг, перейти на главную и вернуться, чтобы показать сохранённое значение фильтра.
3. Перейти в адаптивный режим браузера, менять ширину и объяснить в коде настройки размеров карточек, количества колонок и т.д. (со ссылкой на конкретные значения).
4. На компьютере открыть GitHub Pages и продемонстрировать работу с бэкендом.
5. Показать работу с бэкендом собранного (не dev) приложения Tauri.
6. На компьютере подключить приложение к бэкенду по IP в локальной сети (не `localhost`), сравнить IP сервера из консоли и в коде.
7. С помощью Wireshark или tcpdump найти порт Tauri-приложения.
8. Отредактировать услуги в БД и показать изменения в Tauri.
9. Продемонстрировать HTTPS в локально развернутом фронтенде.

## Контрольные вопросы

- Flux
- Схема Redux (store, reducer, dispatch, action)
- Виды нативных приложений (PWA, Tauri и т.д.)
- GitHub Pages

## Deployment-диаграмма

Нужно изобразить все узлы и компоненты системы: фронтенд, веб-сервер со статикой, веб-сервис, базы данных и другие хранилища. Узлы соединяются протоколами, компоненты фронтенда и бэкенда размещаются внутри соответствующих узлов, указывается API между ними.

## Задание

1. Внедрить менеджер состояний для хранения значений фильтров.
2. Добавить адаптивность и поддержку PWA.
3. Создать Tauri-приложение и развернуть проект в GitHub Pages.

### Подробности

- Использовать Redux Toolkit для хранения состояния фильтра услуг (желательно включить расширение redux-devtools).
- Развернуть фронтенд на GitHub Pages и обеспечить работу в режиме PWA.
- Обеспечить адаптивность для трёх страниц приложения.
- Создать простое Tauri-приложение для интерфейса гостя (без авторизации и редактирования) из трёх страниц с фильтрацией и изображениями.
- Подключить Tauri-приложение к разработанному API через IP-адрес локальной сети (не `localhost`).
